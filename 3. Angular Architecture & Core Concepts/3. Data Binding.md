## **What is Data Binding?**
Data binding is a core concept in Angular that enables communication between the **component (TypeScript class)** and the **view (HTML template)**. It ensures a dynamic and seamless interaction between the UI and the application logic.

### **Types of Data Binding**
Angular supports four types of data binding:
1. **Interpolation (`{{ }}`)** ‚Äì One-way binding from component to view.
2. **Property Binding (`[property]`)** ‚Äì One-way binding from component to element properties.
3. **Event Binding (`(event)`)** ‚Äì One-way binding from UI events to component logic.
4. **Two-Way Binding (`[(ngModel)]`)** ‚Äì Synchronization between the component and view.

---

## **1Ô∏è‚É£ Interpolation (`{{ }}`)**
Interpolation allows embedding **component properties** directly into the template.

### **Example:**
```typescript
export class AppComponent {
  title = 'Angular Data Binding';
}
```

```html
<h1>Welcome to {{ title }}</h1>
```
‚úÖ The **`title`** variable is dynamically displayed in the UI.

üí° **Note:** Interpolation only works with **string values**. For non-strings (e.g., objects, arrays), use property binding.

---

## **2Ô∏è‚É£ Property Binding (`[property]`)**
Property binding binds a **component property** to an **HTML element attribute**.

### **Example:**
```typescript
export class AppComponent {
  imageUrl = 'assets/logo.png';
}
```

```html
<img [src]="imageUrl" alt="Logo">
```
‚úÖ The image source is dynamically set from the **component property**.

üí° **Common Use Cases:**
- Setting `src` for images
- Setting `disabled`, `hidden`, `checked` for form controls

---

## **3Ô∏è‚É£ Event Binding (`(event)`)**
Event binding captures **user actions** (like clicks, keypresses) and triggers methods in the component.

### **Example:**
```typescript
export class AppComponent {
  showMessage() {
    alert('Button Clicked!');
  }
}
```

```html
<button (click)="showMessage()">Click Me</button>
```
‚úÖ When the button is clicked, `showMessage()` is executed.

üí° **Common Use Cases:**
- Handling button clicks
- Capturing form input changes
- Subscribing to keyboard and mouse events

---

## **4Ô∏è‚É£ Two-Way Binding (`[(ngModel)]`)**
Two-way binding synchronizes data **between the component and the view**.

üîπ Requires the `FormsModule` from `@angular/forms`.

### **Example:**
```typescript
import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html'
})
export class AppComponent {
  name: string = '';
}
```

```html
<input [(ngModel)]="name" placeholder="Enter your name">
<p>Hello, {{ name }}</p>
```
‚úÖ The input field and paragraph are synchronized in real time.

üí° **Common Use Cases:**
- Form inputs
- Dynamic search fields
- Live filtering

---

## **Combining Bindings in One Example**
Let's put all types of data binding together in a practical example.

### **Example:**
```typescript
export class AppComponent {
  userName: string = '';
  isDisabled: boolean = false;
  showMessage() {
    alert(`Welcome, ${this.userName}!`);
  }
}
```

```html
<input [(ngModel)]="userName" placeholder="Enter your name">
<button [disabled]="isDisabled" (click)="showMessage()">Submit</button>
<p *ngIf="userName">Hello, {{ userName }}</p>
```
‚úÖ The **input**, **button**, and **paragraph** work dynamically together.

## **Best Practices for Data Binding**
‚úÖ **Use property binding** instead of interpolation for non-string values.
‚úÖ **Avoid using `any` type** in TypeScript for better type safety.
‚úÖ **Use `trackBy` in `*ngFor`** for optimized list rendering.
‚úÖ **Use `OnPush` change detection** when working with large data sets.
‚úÖ **Prefer event binding** over `addEventListener()` for better performance.
